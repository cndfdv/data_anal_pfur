# Frontend: React, вкладки и отображение данных

## Общая роль frontend

Frontend отвечает за две пользовательские задачи:

- поиск статей
- просмотр аналитики

Он не содержит тяжёлой бизнес-логики. Его задача — отправить запрос в backend, получить JSON и показать результат в удобном виде.

## `frontend/src/main.jsx`

Точка входа React-приложения.

Здесь приложение монтируется в DOM-элемент `#root`, который определён в `frontend/index.html`.

## `frontend/src/App.jsx`

Это главный компонент интерфейса.

Он:

- хранит активную вкладку `search` или `analytics`
- хранит текущий текст запроса
- хранит результат поиска
- показывает loader и ошибки

### Состояния в `App`

- `tab` — какая вкладка открыта
- `query` — текущий запрос
- `data` — ответ от `/api/search`
- `loading` — идёт ли поиск
- `error` — ошибка поиска

### Логика `handleSearch`

1. Убирает лишние пробелы.
2. Не отправляет пустой запрос.
3. Вызывает `searchArticles(trimmed, 10)`.
4. Если всё хорошо, кладёт результат в `data`.
5. Если ошибка, показывает сообщение.

## `frontend/src/api.js`

Это маленький слой доступа к backend API.

Он экспортирует три функции:

- `searchArticles`
- `fetchStats`
- `fetchHealth`

Почему это полезно:

- сетевой код вынесен из React-компонентов
- легче менять способ вызова API
- UI-компоненты остаются проще

## `frontend/src/components/SearchBar.jsx`

Компонент поисковой строки.

Что делает:

- хранит локальное значение input
- синхронизирует его с внешним `value`
- при submit вызывает `onSearch(q)`
- умеет работать в двух режимах:
  - большой input в центре стартового экрана
  - компактный input после первого поиска

## `frontend/src/components/ResultCard.jsx`

Карточка одной найденной статьи.

Отображает:

- заголовок
- авторов
- год
- категории
- сокращённый абстракт
- score похожести

### Зачем здесь локальный `expanded`

Если abstract длинный, компонент сначала показывает первые 300 символов.

Кнопка `Show more / Show less` переключает локальное состояние, не затрагивая весь список результатов.

### Как окрашивается score

Функция `scoreColor(score)` выбирает цвет бейджа по диапазону значения.

Это простая визуальная подсказка:

- высокий score — зелёный
- средний — жёлтый
- низкий — серый

## `frontend/src/components/StatsPanel.jsx`

Компонент вкладки аналитики.

При первом рендере он вызывает `fetchStats()`.

Затем показывает:

- summary-карточки
- список top queries
- bar chart по годам
- bar chart по категориям

### Почему здесь свой state

Аналитика живёт отдельно от поиска, поэтому ей не нужен state из `App`.

Это упрощает структуру: вкладка поиска и вкладка аналитики почти независимы друг от друга.

## `frontend/src/components/Loader.jsx`

Skeleton-заглушка для списка результатов.

Нужна, чтобы интерфейс не "прыгал" во время ожидания ответа.

## Vite и proxy

### `frontend/vite.config.js`

Используется Vite dev server и proxy для `/api`.

Зачем proxy нужен:

- frontend можно открывать на `:5173`
- backend работает на `:8000`
- браузеру не нужно напрямую знать адрес backend

В Docker-окружении proxy target должен смотреть на `http://backend:8000`, потому что `backend` — это имя сервиса в сети Compose.

## Почему frontend сделан именно так

Для учебного проекта выбран простой подход:

- немного state
- без Redux и без React Query
- минимум абстракций
- понятное разделение на мелкие компоненты

Это делает код удобным для объяснения на защите: можно быстро показать, где начинается поиск, где рисуются карточки и откуда берётся аналитика.
