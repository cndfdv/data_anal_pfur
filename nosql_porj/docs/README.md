# Документация по коду

Эта папка нужна как "объяснялка" по проекту: что здесь есть, как части связаны между собой и зачем нужен каждый файл.

## Что читать в каком порядке

1. [01-overview.md](./01-overview.md)  
   Общая идея проекта, роли всех сервисов и ключевой сценарий работы.

2. [02-infrastructure.md](./02-infrastructure.md)  
   Docker Compose, контейнеры, переменные окружения и загрузка данных.

3. [03-backend.md](./03-backend.md)  
   Подробное объяснение FastAPI, клиентских модулей БД и поиска.

4. [04-frontend.md](./04-frontend.md)  
   Как устроен React-интерфейс, вкладки, запросы к API и отображение результатов.

5. [05-file-map.md](./05-file-map.md)  
   Объяснение почти каждого файла в проекте: что он делает и за что отвечает.

## Для чего это полезно

- Быстро подготовиться к защите проекта
- Понять архитектуру без чтения всего кода подряд
- Найти нужный модуль и понять, где искать баг или место для доработки
- Объяснить преподавателю, почему здесь несколько NoSQL БД и как они используются вместе
